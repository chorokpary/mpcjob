# IP 접근 제한 설정 (`ip_list.asp`) 사용 매뉴얼

본 문서는 관리자 접속 IP 접근 제한 설정 페이지([ip_list.asp](file:///d:/MPCJOB/mpcjob_2012/admin/iplimit/ip_list.asp))의 기능, 동작 원리 및 관리자 사용법을 정리한 매뉴얼입니다.

## 1. 📌 개요

* **페이지 경로**: `/admin/iplimit/ip_list.asp`
* **목적**: 허용된 IP 주소 대역에서만 관리자 페이지에 접근할 수 있도록 제한하여 비인가 사용자의 무단 접속을 차단하고 보안을 강화합니다.
* **관련 주요 파일 및 DB**:
  | 구분 | 파일/테이블명 | 설명 |
  | :--- | :--- | :--- |
  | **메인 설정 페이지** | [ip_list.asp](file:///d:/MPCJOB/mpcjob_2012/admin/iplimit/ip_list.asp) | 허용 IP 목록 조회, 신규 IP 등록, 사용 여부 토글, 삭제 및 예외 ID 팝업 호출 |
  | **IP 처리 모듈** | [ip_proc.asp](file:///d:/MPCJOB/mpcjob_2012/admin/iplimit/ip_proc.asp) | IP 등록(`insert`), 삭제(`delete`), 사용상태 변경(`toggle`) DB 처리 |

  | **예외 ID 팝업** | [ip_bypass_pop.asp](file:///d:/MPCJOB/mpcjob_2012/admin/iplimit/ip_bypass_pop.asp) | IP 제한을 받지 않는 예외 관리자 계정 ID 등록/삭제 팝업 |

  | **예외 ID 테이블** | `TBL_IP_BYPASS` | IP 차단 대상에서 무조건 예외 처리(통과)되는 관리자 ID 저장 |

## 2. 🛡️ IP 접근 제한 동작 방식 (보안 로직)

1. **락아웃(Lockout) 방지 안전 장치**
   * `TBL_IP_ALLOW` 테이블에 **`IsUse = 'Y'`(사용 중) 상태의 허용 IP가 1개 이상 등록되어 있을 때만** IP 차단 기능이 활성화됩니다.
   * **등록된 활성 IP가 0개인 경우**: 관리자 실수로 인한 접속 불가 사태를 방지하기 위해 IP 제한이 작동하지 않으며 모든 IP에서 접속이 허용됩니다.

2. **접속 제어 3단계 검사 흐름**
   ```
   [사용자 페이지 접속 / 로그인 시도]
             │
             ▼
   [1단계: 예외 ID 검사] ──(TBL_IP_BYPASS에 등록된 ID인가?)──► [YES] ──► 접속 허용 (Pass)
             │ [NO]
             ▼
   [2단계: 활성 IP 존재 여부] ──(TBL_IP_ALLOW 사용 중 IP가 0개인가?)──► [YES] ──► 접속 허용 (Pass)
             │ [NO (1개 이상 존재)]
             ▼
   [3단계: IP 허용 여부 검사] ──(접속 IP가 TBL_IP_ALLOW에 등록되어 있는가?)
             ├──────► [YES] ──► 접속 허용 (Pass)
             └──────► [NO]  ──► 차단 경고창(Alert) 출력 후 로그인 페이지로 이동 (Block)
   ```

## 3. 🖥️ 화면 기능 및 사용 방법

### ① 상단 접속 정보 및 편의 기능
* **현재 접속 IP 표시**: 현재 접속 중인 사용자의 IP 주소(`REMOTE_ADDR`)를 실시간으로 보여줍니다.
* **`내 IP 자동입력` 버튼**: 클릭 시 현재 접속 IP가 하단 "허용 IP 주소" 입력 칸에 자동으로 입력되어 편리하게 등록할 수 있습니다.
* **`🔓 예외 ID 관리` 버튼**: IP 제한을 받지 않는 예외 관리자 계정을 등록/삭제할 수 있는 팝업창([ip_bypass_pop.asp](file:///d:/MPCJOB/mpcjob_2012/admin/iplimit/ip_bypass_pop.asp))을 엽니다.

### ② 허용 IP 신규 등록
1. **허용 IP 주소**:
   * **단일 IP**: 예) `210.112.73.45`
   * **와일드카드(`*`) 대역 지정**: 예) `210.112.73.*` (입력 시 `210.112.73.1` ~ `210.112.73.254` 전체 대역 허용)
2. **식별 메모**: 해당 IP의 위치나 용도(예: `본사 사무실`, `개발팀 락커룸 IP` 등)를 입력합니다.
3. **`허용 IP 등록` 버튼**: 클릭 시 `TBL_IP_ALLOW` 테이블에 `IsUse = 'Y'` 상태로 저장되며 즉시 적용됩니다.

### ③ 허용 IP 목록 관리 (테이블)
* **사용 상태 토글 (`사용 중` / `일시 중지`)**:
  * **`사용 중` (초록색)**: 해당 IP 제한 정책이 적용 중입니다.
  * **`일시 중지` (빨간색)**: 삭제하지 않고 임시로 해당 IP의 허용 상태를 비활성화합니다.
  * 클릭 시 `Y` $\leftrightarrow$ `N` 상태가 즉시 전환됩니다.
* **삭제**: 더 이상 사용하지 않는 IP 항목을 클릭하여 DB에서 완전히 제거합니다.

## 4. 🔓 예외 관리자 ID 관리 (`ip_bypass_pop.asp`)

IP 접근 제한이 설정된 환경에서 재택근무, 외부 출장 등 허용되지 않은 IP 주소로 로그인해야 하는 관리자를 위해 특정 **계정 ID 단위로 IP 차단을 무시(Bypass)**할 수 있습니다.

1. `ip_list.asp` 상단의 **`🔓 예외 ID 관리`** 버튼 클릭
2. 팝업창에서 예외 처리할 **관리자 ID**와 **구분 메모** 입력 후 `등록`
3. 등록된 ID로 로그인하는 사용자는 IP 주소 검사를 통과하여 제한 없이 접속 가능

## 5. ⚠️ 주의사항 및 FAQ

1. **IP를 등록했는데 여전히 차단되는 경우**
   * 등록한 IP의 상태가 `일시 중지`로 되어 있는지 확인하십시오. (`사용 중` 상태여야 함)
   * 와일드카드(`*`) 사용 시 공백이나 오타가 없는지 확인하십시오.

2. **유동 IP 환경인 경우 관리 방법**
   * 와일드카드(`*`)를 사용하여 대역 전체(예: `221.148.11.*`)를 등록하거나,
   * 담당자 계정 ID를 **예외 관리자 ID (`TBL_IP_BYPASS`)**에 등록하여 관리하는 것을 권장합니다.

3. **최초 설정 시 주의사항**
   * 반드시 **[내 IP 자동입력]**을 눌러 관리자 본인의 현재 IP를 먼저 등록한 후에 다른 정책들을 추가/관리하십시오.
